# Escaneos genómicos
Con los escaneos genómicos podemos explorar parámetros a lo largo de un genoma a fin de encontrar regiones con valores extremos. Cuando buscamos posibles señales de selección, se recomienda no excluir regiones codificantes o reguladoras. Por otro lado, si se recomienda incluir solo regione de alta calidad, obviando regiones con bajos puntajes de mapeo, profundidad de secuenciación excesivamente baja/alta, y regiones de baja complejidad que puedan confundir los análisis.

## Estimación de 2d-SFS
El espectro de frecuencia de sitios de dos dimensiones (2d SFS) es una extensión del 1d SFS para dos poblaciones. Primero, se obtiene el SFS para cada población. Creamos un script para someter al HPC con `nano 1-1dsfs`.
```bash
!/bin/bash

# Nombre del proceso:
#SBATCH --job-name=1-1dsfs

# Proyecto:
#SBATCH --account=tipo-iii
#SBATCH --partition=standard # Nombre de la partición

# Recursos:
#SBATCH --time=24:00:00 # Límite de tiempo de corrida
#SBATCH --cpus-per-task=15 # Límite de CPUs para cada tarea en tareas de múltiples instancias
#SBATCH --mem=25G # Límite de memoria

# Archivos de salida
#SBATCH --output=/data/courses/course2026/USERNAME/logs/1dsfs_o%j # Nombre del archivo de salida stdout
#SBATCH --error=/data/courses/course2026/USERNAME/logs/1dsfs_e%j # Nombre del archivo de error stderr

# Ambiente de trabajo
module load angsd
module list

# Mensaje informativo
echo "Tarea ${SLURM_JOB_ID} name ${SLURM_JOB_NAME} iniciado en:"
date
echo "Nodo:"
hostname

# Comandos a ejecutar
angsd_prefix="NORTH"
ref_genome="/data/courses/course2026/course_files/mapeo_variantes/ppa_v1.asm.fasta"
bam_list="/data/courses/course2026/course_files/escaneos_genomicos/NORTH.list"
threads=10
nind=$(wc -l $bam_list | cut -d' ' -f1)
let minind=$nind/2
callable_sites="/data/courses/course2026/course_files/escaneos_genomicos/NORTH.callable_sites.bed"

echo "Crear archivo de sitios válidos en formato de angsd a partir de un archivo bed"
date
cut -f1-3 $callable_sites | awk '{print $1 "\t" $2+1 "\t" $3}' > $angsd_prefix.angsdfile
angsd sites index $angsd_prefix.angsdfile

seq 1 14 | awk '{print "Chr" $1}' > Chrs.txt

echo "Estimar las probabilidades de genotipo para el grupo $angsd_prefix"
date
angsd -GL 2 -trim 5 -doMajorMinor 1 -doGlf 2 -doSaf 1 -doCounts 1 -nThreads $threads \
 -minMapQ 30 -minQ 20 -setMinDepthInd 2 -minInd $minind -remove_bards 1 -uniqueOnly 1 \
 -baq 1 -C 50 -ref $ref_genome -anc $ref_genome -b $bam_list \
  -rf Chrs.txt -sites $angsd_prefix.angsdfile -out $angsd_prefix

echo "Obtener el SFS 1d para el grupo $angsd_prefix"
date
realSFS $angsd_prefix.saf.idx -P $threads -fold 1 -seed 1 > $angsd_prefix.sfs

echo "Obtener indices de diversidad genetica por segmentos y por ventanas para $angsd_prefix"
date
realSFS saf2theta $angsd_prefix.saf.idx -outname $angsd_prefix.saf2theta -sfs $angsd_prefix.sfs -P $threads -fold 1 -seed 1
thetaStat do_stat $angsd_prefix.saf2theta.thetas.idx -outnames $angsd_prefix.saf2theta.thetas.idx.chrs
thetaStat do_stat $angsd_prefix.saf2theta.thetas.idx -win 50000 -step 10000 -outnames $angsd_prefix.saf2theta.thetas.idx.w50k.s10k

echo "Job finalizado en:"
date
```
Sometemos el proceso para los _n_ grupos en evaluación. Asegurarse de cambiar el `USERNAME` por el nombre de usuario. Tambien, cambiar los archivos para cada grupo en especifico ("NORTH", "SOUTH")
```bash
sbatch 1-1dsfs.sh
```
De manera similar al análisis de cada población, el 2d SFS nos permite estimar parámetros de diferenciacion genética entre poblaciones, tanto a escala global como en ventanas. Para ello, construimos el 2d SFS para comparar los grupos en pares. Comenzar creando el script con `nano 2-2dsfs.sh`:
```bash
!/bin/bash

# Nombre del proceso:
#SBATCH --job-name=2-2dsfs

# Proyecto:
#SBATCH --account=tipo-iii
#SBATCH --partition=standard # Nombre de la partición

# Recursos:
#SBATCH --time=24:00:00 # Límite de tiempo de corrida
#SBATCH --cpus-per-task=15 # Límite de CPUs para cada tarea en tareas de múltiples instancias
#SBATCH --mem=25G # Límite de memoria

# Archivos de salida
#SBATCH --output=/data/courses/course2026/USERNAME/logs/2dsfs_o%j # Nombre del archivo de salida stdout
#SBATCH --error=/data/courses/course2026/USERNAME/logs/2dsfs_e%j # Nombre del archivo de error stderr

# Ambiente de trabajo
module load angsd
module list

# Mensaje informativo
echo "Tarea ${SLURM_JOB_ID} name ${SLURM_JOB_NAME} iniciado en:"
date
echo "Nodo:"
hostname

# Comandos a ejecutar
angsd_prefix="NORTH.SOUTH"
ref_genome="/data/courses/course2026/course_files/mapeo_variantes/ppa_v1.asm.fasta"
group1_saf="/data/courses/course2026/course_files/escaneos_genomicos/NORTH.saf.idx"
group2_saf="/data/courses/course2026/course_files/escaneos_genomicos/SOUTH.saf.idx"
threads=10

echo "Obtener el SFS 2d para los grupos $angsd_prefix"
date
realSFS $group1_saf $group2_saf -P $threads -seed 1 > $angsd_prefix.sfs

echo "Obtener indices de diferenciacion genetica por segmentos y por ventanas para $angsd_prefix"
date
realSFS fst index $group1_saf $group2_saf $angsd_prefix.sfs -fold 1 -fstout $angsd_prefix
realSFS fst stats $angsd_prefix.fst.idx > $angsd_prefix.stats
realSFS fst stats2 $angsd_prefix.fst.idx -win 50000 -step 10000 > $angsd_prefix.w50k.s10k.stats

echo "Job finalizado en:"
date
```
Sometemos el proceso al organizador. Recordar cambiar `USERNAME` por el nombre de usuario. También se puede modificar el tamaño y sobrelape de las ventanas en el análisis por ventanas.
```bash
sbatch 2-2dsfs.sh
```
El análisis por ventanas nos permite comparar la tendencia general (_background_) con casos anómalos locales (_outliers_).

## Análisis de enriquecimiento funcional de genes
Un análisis de enriquecimiento funcional de genes se refiere a la exploración de patrones biológicos sobrerrepresentados en genes seleccionados bajo algún criterio a lo largo del genoma. En este caso, el criterio es que se encuentren cerca a alguna señal de selección, por lo que el análisis nos permitirá explorar las tendencias de mecanismos moleculares y procesos biológicos que podrían estar bajo selección.

```r
### Packages
library(topGO)
### Working dir
setwd("/path/to/working_directory/")

## Upload data
significant_genes <- "significant_list.txt"
geneID2GO <- readMappings(file = "gene_terms.txt")
GO2geneID <- inverseList(geneID2GO)
geneNames <- names(geneID2GO)

## Gene functional enrichment analysis
Significant <- read.table(file = significant_genes, sep = "\t")
Significant <- as.character(Significant[,1])
geneList <- factor(as.integer(geneNames %in% Significant))
names(geneList) <- geneNames

# BP terms
BPGOdata <- new("topGOdata", ontology = "BP", allGenes = geneList, annot = annFUN.gene2GO, gene2GO = geneID2GO, nodeSize = 10)
resultBPFisher <- runTest(BPGOdata, algorithm = "weight01", statistic = "fisher")
# resultBPFisher # Put the number of GO terms in topNodes to show all results
allBPRes <- GenTable(BPGOdata, classicFisher = resultBPFisher, orderBy = "classicFisher", ranksOf = "classicFisher", topNodes = 30)
write.table(allBPRes, file = "BP_Table_Significant.txt", sep = "\t", quote = FALSE)

# MF terms
MFGOdata <- new("topGOdata", ontology = "MF", allGenes = geneList, annot = annFUN.gene2GO, gene2GO = geneID2GO, nodeSize = 10)
resultMFFisher <- runTest(MFGOdata, algorithm = "weight01", statistic = "fisher")
# resultMFFisher # Put the number of GO terms in topNodes to show all results
allMFRes <- GenTable(MFGOdata, classicFisher = resultMFFisher, orderBy = "classicFisher", ranksOf = "classicFisher", topNodes = 30)
write.table(allMFRes, file = "MF_Table_Significant.txt", sep = "\t", quote = FALSE)

# CC terms
CCGOdata <- new("topGOdata", ontology = "CC", allGenes = geneList, annot = annFUN.gene2GO, gene2GO = geneID2GO, nodeSize = 10)
resultCCFisher <- runTest(CCGOdata, algorithm = "weight01", statistic = "fisher")
# resultCCFisher # Put the number of GO terms in topNodes to show all results
allCCRes <- GenTable(CCGOdata, classicFisher = resultCCFisher, orderBy = "classicFisher", ranksOf = "classicFisher", topNodes = 30)
write.table(allCCRes, file = "CC_Table_Significant.txt", sep = "\t", quote = FALSE)
```
